# Operator Clarity Checklist

Use this checklist to judge whether FST UI works for real DIT/Data Wrangler operation.

## 3-Second Read Test

Within 3 seconds, the operator should know:

- [ ] What phase the job is in.
- [ ] Whether copy is active, verify is active, or job is complete.
- [ ] Whether any warning/error exists.
- [ ] Whether the job is SAFE TO EJECT.
- [ ] What source is being copied.
- [ ] What destination is being written.
- [ ] Approximate whole-job time remaining if available.

## Producer Communication Test

The UI should help answer:

- [ ] How much longer for the whole job?
- [ ] Is copy done?
- [ ] Is verify done?
- [ ] Is it safe to eject?
- [ ] Is there a report?

## Failure Clarity Test

When something fails:

- [ ] The failed phase is obvious.
- [ ] The reason is visible if known.
- [ ] SAFE TO EJECT is clearly NO.
- [ ] Report status is clear.
- [ ] Operator knows next action.

## Anti-Confusion Test

Reject UI if:

- [ ] Copy complete looks like verified complete.
- [ ] Per-file ETA looks like project ETA.
- [ ] Warning is visually subtle.
- [ ] Error is hidden.
- [ ] Success and blocked states look too similar.
- [ ] Technical detail overwhelms primary status.
