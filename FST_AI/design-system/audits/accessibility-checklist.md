# FST Accessibility Checklist

Accessibility in FST is a safety feature.

Operators need to read state quickly and accurately under pressure.

## Contrast

- [ ] Critical text is readable.
- [ ] Warning/error text has strong contrast.
- [ ] Disabled controls are still understandable.
- [ ] Safety status is not low contrast.

## Non-Color State

- [ ] SAFE TO EJECT status uses text.
- [ ] Error state uses text.
- [ ] Warning state uses text.
- [ ] Verify/copy states use labels, not color alone.

## Motion

- [ ] Reduced motion is respected.
- [ ] Progress animation is not distracting.
- [ ] Motion does not hide state changes.

## Keyboard / Focus

- [ ] Important controls have focus states.
- [ ] Cancel remains reachable when appropriate.
- [ ] Report action remains reachable when available.

## Text

- [ ] Critical labels are not too small.
- [ ] Paths are readable or expandable.
- [ ] ETA labels specify Project ETA / Whole Job ETA.
- [ ] Error wording is specific.
