## Release Focus

v1.2 is the Runtime Copy Progress / Operator Progress release.

User-visible improvements:

- Copy UI shows elapsed time, current item, copied bytes, total bytes, copied files, total files, ETA, current speed, and a large Copy Progress bar.
- Verify UI shows current verify file and a large Verify Progress bar.
- Technical Logs retain rsync diagnostics and destination observer diagnostics.
- If rsync progress output is delayed, the UI can still show that destination activity is ongoing.

## Safety Model

Three truth layers must remain separate:

- Safety truth: verification result, report generation, and SAFE TO EJECT.
- Transfer truth: bundled rsync 3.4.4 lifecycle, exit status, errors, and cancellation.
- Operator truth: destination observer UI metrics for copied bytes, copied files, current item, speed, elapsed time, and ETA.

Non-negotiable rule:

Destination observer metrics are UI-only estimates. They must never mark copy success, verification success, report success, or SAFE TO EJECT.

## Runtime QA

- [ ] Transfer starts and enters TRANSFERRING/COPYING without UI freeze.
- [ ] Technical Logs show rsync output or observer activity quickly after copy starts.
- [ ] Copy Progress moves from 0% early when destination activity is observed.
- [ ] Current item, copied bytes/files, speed, elapsed time, and ETA update during copy when available.
- [ ] Verify Progress shows current verify file and progress during verification.
- [ ] SAFE TO EJECT appears only after copy success and verification pass.
- [ ] Verification `none` ends at transfer complete/copy complete, not SAFE TO EJECT.
- [ ] Cancelled, failed, incomplete, or uncertain jobs never show SAFE TO EJECT.
- [ ] TXT report is saved and matches final job state.
- [ ] Operator-facing text uses SAFE TO EJECT, not formatting language.

## Known Limitations

- rsync stdout can arrive late or in bursts because app pipes are not terminals.
- Destination observer improves visibility during rsync-silent windows but does not prove copy success.
- ETA is an observed estimate, not a safety guarantee.
- Current item can show APFS or rsync partial filenames.
- Multi-destination remains outside MVP scope.
